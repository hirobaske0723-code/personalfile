# 専門エージェント型 仮想カンパニー設計

## 概要
現在の `/company` スキルは Claude 1体が全部署を担うが、
部署ごとに専門エージェントを分けてマルチエージェント構造にする。

## 課題・背景
- 今は1体が全部やるので、専門性が薄い
- 調査・分析・記録が直列になっている
- コンテキストが混在して精度が落ちる場合がある

## アーキテクチャ

```
秘書（オーケストレーター）
  ├── TODO/メモ/壁打ち/inbox → 秘書が直接処理
  ├── リサーチエージェント   ← 調査・情報収集に特化
  ├── マーケエージェント     ← コンテンツ・SNS戦略に特化
  └── エンジニアリングエージェント ← 技術ドキュメント・設計に特化
```

## 設計決定事項（2026-03-22）

### エージェント化する部署
- リサーチ・マーケティング・エンジニアリング
- PM・経理・営業・クリエイティブ・人事は使い始めてから追加検討

### 品質方針
- 迷ったら専門エージェントに投げる（品質優先）
- 秘書が直接処理するのは TODO・メモ・壁打ち・inbox のみ

### 複数部署またがる場合
- 依存あり → 直列（リサーチ結果をマーケに渡す等）
- 依存なし → 並列（エンジニアリング + マーケ等）

### エージェントプロンプト構造
```
【常に渡す】
- 部署の CLAUDE.md（役割・ルール）
- ユーザーの依頼内容
- 保存先パス
- オーナーコンテキスト（.company/CLAUDE.md のパーソナライズメモ）

【直列時のみ追加】
- 前のエージェントの出力結果
```

### 実装方針
- まずプロジェクトルートに試作スクリプトを作る
- 動作確認後に /company スキルへ組み込む
